# Project Atlas Release Strategy

## Versioning

Project Atlas follows Semantic Versioning.

MAJOR.MINOR.PATCH

Example:

1.0.0

## Major

Large architectural changes.

## Minor

New capabilities.

## Patch

Bug fixes and maintenance.

---

## Development

0.x.x

Feature development.

---

## Production

1.x.x

Stable releases.

---

## Release Gates

A release requires:

- Engineering review
- Handbook updated
- Build log updated
- ADRs updated
- Clean Git tree
- atlas doctor passes
